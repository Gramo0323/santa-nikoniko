# A3: 証拠 (debug_helpPoints)

## ブラウザConsoleログ
ブラウザ起動直後に「+1」ボタンをクリックした際、以下の警告が出力されることを確認。
```
Skipping save: Not hydrated yet.
```
これにより、初期ロード完了前の操作が保存されないことが証明された。

## localStorageの挙動
1. ページロード時: `santa_help_total` は `15`。
2. クリック3回実施: UI上はゲージが進むが、localStorageは `15` のまま。
3. リロード実施: `helpTotal` が `15` に戻る（3回分が消失）。

## 画面表示の証拠
- 初期表示: 「スタンプ: 3こ」「あと5かい」
- クリック後: 「スタンプ: 3こ」「あと2かい」
- リロード後: 「スタンプ: 3こ」「あと5かい」
（リロードによって進捗が破棄されている）

## 原因の切り分け
- クリックハンドラ: 動作している（UIとメモリ上の `helpTotal` は更新される）。
- 保存処理: `isHydrated` が `false` のため、 `saveData()` 内でリターンされる。
- 復元処理: `loadData()` で古い値が読み込まれ、メモリ上の `helpTotal` を上書きする。
