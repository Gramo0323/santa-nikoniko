# A1: 修正仕様 (fix_helpPoints)

## 修正の目的
「おてつだい」ボタンの操作が保存されない、およびロード完了前の操作でデータが上書きされる不具合を修正する。

## コミット1：根因1の修正
- **目的**: ログイン中のロード完了時に `isHydrated` フラグを確実に `true` にし、保存処理を有効化する。
- **DoD (Definition of Done)**:
  - `loadDataFromSupabase` の正常系および異常系（finally）で `isHydrated = true` が設定されている。
  - ログイン状態でボタンを押した後、リロードしても値が保持されている。

## コミット2：根因2の修正
- **目的**: データロード中（`isHydrated = false`）の操作を禁止し、データの不整合を防ぐ。
- **DoD (Definition of Done)**:
  - ページ読み込み直後、ボタンが `disabled` である。
  - `isHydrated` が `true` になった瞬間にボタンが有効化される。
  - ロード中のクリックが無視されるガードロジックが追加されている。
